## 프롬프트 유형과 선택 이유
```
프롬프트 유형: 페르소나 기반 콘텐츠 변환 (Persona-based Content Transformation) 프롬프트 (또는 롤-태스크-포맷 프롬프트)

선택 이유:

페르소나 (Persona): '숏폼 콘텐츠 PD'라는 명확한 **역할(Role)**을 AI에 부여합니다. 이는 단순 텍스트 요약자가 아닌, '영상 문법(시각적 요소, 효과음)'과 '시청자 후킹(초반 3초)'을 우선적으로 고려하는 전문가의 관점에서 결과물을 생성하도록 유도하기 위함입니다. (참조 자료의 '페르소나' 유형 활용)

콘텐츠 변환 (Transformation): '10페이지 분량의 전문 리포트(Input)'를 '초등학생 수준의 60초 영상 대본(Output)'으로 바꾸는 것은 단순 요약이 아닌, **매체 변환(Media Transformation)**입니다. '딱딱한 텍스트'를 '시각적 스크립트'라는 완전히 다른 형식으로 변환하는 것이 핵심 과제입니다. (힌트의 '매체 변환' 및 참조 자료의 '형식 변환' 유형 활용)

구체적 제약 (Constraints): 'JSON 형식', '60초 분량', '초반 3초 후킹 멘트' 등 명확한 제약 조건을 포함시켜, AI가 주어진 목표에 정확히 부합하는 창의적이면서도 통제된 결과물을 생성하도록 지시합니다.
```

## 작성 위해 추가 필요 요건
```
이 과제를 완수하는 프롬프트를 작성하기 위해, '숏폼 콘텐츠 PD'로서 AI에게 제공해야 할 추가 요건들입니다.

[필수] 원본 리포트 텍스트: 변환 대상인 "생성형 AI의 미래와 윤리적 쟁점" 10페이지 리포트의 전체 텍스트가 입력값으로 반드시 필요합니다.

[핵심] 마케팅 팀의 핵심 메시지 (Key Takeaway): 10페이지 전체를 60초에 담는 것은 불가능합니다. 마케팅 팀이 이 리포트를 통해 시청자에게 전달하고자 하는 가장 중요한 단 하나의 핵심 메시지 또는 강조하고 싶은 2~3가지 키워드가 필요합니다.

[톤앤매너] 채널 레퍼런스: '테크인사이드' 채널의 기존 '60초 지식 요약' 시리즈 중 가장 반응이 좋았던 영상의 대본 예시 또는 채널의 구체적인 톤앤매너 (예: 유머러스, 진지함, 속도감 등)가 필요합니다.

[타겟] 후킹을 위한 타겟 명확화: '대중'이나 '초등학생'은 너무 광범위합니다. 틱톡/릴스에서 이 주제를 소비할 핵심 타겟 페르소나 (예: AI에 막연한 불안감을 가진 20대, AI 그림 그리기에만 관심 있는 10대 등)를 구체화해야 3초 후킹 멘트를 날카롭게 만들 수 있습니다.
```

## 가상 문제 상황
```
1. 가상 문제 상황
👤 페르소나: 뉴미디어 스타트업의 '숏폼 콘텐츠 PD'
🏙️ 배경: 당신은 IT 및 테크 트렌드를 소개하는 유튜브 채널을 운영하는 스타트업 '테크인사이드'의 PD입니다. 최근 회사는 틱톡과 릴스 유입을 늘리기 위해 '60초 지식 요약' 시리즈를 런칭했습니다.
🚨 핵심 문제: 마케팅 팀으로부터 **"생성형 AI의 미래와 윤리적 쟁점"**이라는 주제의 10페이지짜리 심층 분석 리포트(PDF)를 전달받았습니다. 내용은 매우 전문적이고 딱딱한 텍스트 위주여서, 이를 그대로 읽으면 시청 지속 시간이 3초 미만으로 떨어질 것이 확실시되는 상황입니다.
🎯 해결 과제:
복잡하고 긴 텍스트 리포트 내용을 대중이 이해하기 쉬운 '초등학생도 이해하는 수준'의 언어로 변환해야 합니다.
단순 요약을 넘어, 시각적 요소(화면 구성, 자막, 효과음)가 포함된 **60초 분량의 숏폼 영상 촬영 대본(Script)**을 완성해야 합니다. (특히, 초반 3초 안에 시선을 끄는 '후킹(Hooking)' 멘트가 필수입니다.)
출력은 JSON 형식
2. 훈련 초점 (힌트)
이 문제는 [복잡한 텍스트의 요약 및 매체 변환(Text-to-Script)] 능력을 평가
단순히 내용을 줄이는 것이 아니라, **영상 문법(Visual Cues)**에 맞게 내용을 재구성하는 프롬프트 엔지니어링이 필요.
```

## 문제 해결 프롬프트
```
[ROLE] 당신은 30년차 경력의 숏폼 콘텐츠 PD, 생성형 AI 전문가, 콘텐츠 작가, 모션 그래픽 디자이너, 마케팅 담당자입니다.

지식 번역: '생성형 AI'와 같은 극도로 복잡하고 딱딱한 주제를 '초등학생도 이해하는 수준'의 언어로 즉시 변환할 수 있습니다.

숏폼 문법: 틱톡/릴스 시청자의 3초 이탈률을 정확히 이해하며, 스크롤을 즉시 멈추게 하는 '후킹 멘트'의 대가입니다.

시각적 스토리텔링: 텍스트를 단순 나열하는 것이 아니라, 시각적 요소(모션 그래픽, 자막, 효과음)가 결합된 완전한 '영상 대본'으로 기획할 수 있습니다.

[TASK] 아래 [원본 리포트]의 핵심 내용을 기반으로, 틱톡/릴스 플랫폼에 최적화된 '60초 분량의 숏폼 영상 대본' 1개를 [출력 형식]에 맞춰 생성합니다.

[INPUT: 원본 리포트] https://drive.google.com/file/d/1VwrpqxZccttzvMLFhXo-4qzZKSY9yf05/view?usp=sharing

[CRITICAL CONSTRAINTS & GUIDELINES]

초강력 3초 후크 (The Hook):

대본의 첫 3초(약 15자 내외)는 시청자의 스크롤을 즉시 멈출 수 있는 가장 강력한 질문이나 충격적인 팩트로 구성해야 합니다.

절대 리포트의 제목("생성형 AI의 미래...")을 그대로 사용하지 마십시오. (예: "AI가 당신을 차별한다면?", "AI 때문에 내일 잘릴 사람?")

극한의 단순화 (Simplification):

'윤리적 쟁점', '편향성', '사회적 불평등' 같은 모든 전문 용어는 반드시 비유나 일상적인 예시로 대체해야 합니다.

(예: '편향성' -> "AI가 특정 그룹만 싫어하게 배우는 것", '윤리적 쟁점' -> "이거 써도 되나? 싶은 고민")

핵심 메시지 1개 집중 (One Message):

10페이지 리포트의 모든 내용을 담으려 하지 마십시오. 60초 안에 임팩트를 줄 수 있는 가장 흥미롭거나 충격적인 단 하나의 핵심 메시지에만 집중해야 합니다.

시각적 대본 (Visual Script):

단순 나레이션이 아닌, [시각적 연출]과 [효과음] 지시사항이 포함된 대본이어야 합니다. AI 전문가의 말이 아닌, 모션 그래픽 디자이너의 관점에서 텍스트를 시각화해야 합니다.

속도감 (Pacing):

전체 나레이션은 60초 이내에 명확하게 들릴 수 있도록 속도감 있게 구성되어야 합니다. (약 150~170 단어 내외)

[OUTPUT FORMAT: JSON] 결과물은 반드시 다음의 구조를 가진 JSON 형식으로만 출력해야 합니다. 각 장면(Scene)은 시각적 연출(visuals)과 효과음(sfx)을 포함해야 합니다.
```
